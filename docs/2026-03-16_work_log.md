## 3. Historical validation and model review

- Ran season-based team-model benchmarks across the rebuilt tournament dataset.
- Trained and selected the best matchup model for bracket simulation.
- Ran bracket backtests across all usable tournament seasons:
  - `2011-2019`, `2021-2025`
- Confirmed `2015` is no longer skipped in the backtest.
- Reviewed seed impact:
  - seed is not uniformly helpful for every team-level task
  - seed helps much more clearly in the matchup model
- Reviewed feature importance directionally:
  - `AdjEM` is the strongest signal
  - `AdjO` and `AdjD` are next-tier core signals
  - rebounding, turnovers, and shooting quality are useful second-tier screening factors
- Reviewed model behavior by round and by bracket path:
  - actual game winner prediction is solid across rounds
  - exact late-round field prediction is much harder than early rounds

## 8. Streamlit Community Cloud deployment issue

- Reviewed the first deployment log from Streamlit Community Cloud.
- Identified that the failure was environmental, not application logic:
  - the app was deployed under Python `3.14.3`
  - pinned versions of `pandas==2.1.4` and `scipy==1.11.4` do not build cleanly there in Streamlit's environment
- Recommended redeploying with Python `3.11` or `3.12`, preferably `3.11` to match local work more closely.
